# Career Roadmap for a 2028 CS Grad 🎯

Let me be honest with you, as a senior engineer would be.

---

## The Brutal Truth First

You're right that AI will handle a lot of code generation by 2028. But here's what most people get wrong — **AI amplifies capable engineers, it replaces helpless ones.** The question isn't "will AI take over?" — it's "will she be someone who *directs* AI or someone AI *replaces*?"

The engineers who will thrive are those who understand what the AI is building, can spot when it's wrong, and know how to ask the right questions.

---

## The Mindset Shift

Stop thinking **"learn to code"** → Start thinking **"learn to engineer systems"**

There's a massive difference. Coding is typing. Engineering is thinking.

---

## The Roadmap (Honest & Realistic)

### Phase 1 — Foundations (Now → 6 months)
This cannot be skipped. Ever.

**DSA (Daily, non-negotiable)**
- Arrays, Strings, LinkedLists, Stacks, Queues
- Trees, Graphs, Recursion
- Dynamic Programming (this separates juniors from seniors)
- Practice on LeetCode — easy first, then medium

**Web Fundamentals (2-3 months)**
- HTML/CSS — not deeply, just enough to read and understand UI
- JavaScript — this one matters more. Promises, async/await, closures, event loop
- Don't go framework-first. Understand the language first.

**One Framework (pick React)**
- Build 3-4 real projects, not tutorials
- A todo app teaches nothing. Build something she'd actually use.

---

### Phase 2 — The Real Differentiator (6–12 months)
This is where most students stop. This is exactly where she should *start* going deeper.

**Systems Thinking**
- How does the internet actually work? DNS, HTTP, TCP/IP
- What happens when you type google.com? She should be able to explain this completely
- Databases — SQL deeply, then touch NoSQL
- How do APIs work, REST vs GraphQL

**One Backend Language (Python or Node.js)**
- Build APIs, connect databases, handle authentication
- Deploy something real — not just localhost

**Cloud Basics (AWS or GCP)**
- EC2, S3, basic deployment
- This is table stakes by 2028

---

### Phase 3 — The 2028 Edge (12–24 months)
This is what will separate her from 10,000 other CS grads.

**AI/ML Literacy — NOT optional by 2028**
- She doesn't need to build models. She needs to *understand* them.
- How LLMs work conceptually — tokens, context windows, embeddings
- How to use APIs — OpenAI, Anthropic, Gemini
- **Prompt engineering is a real engineering skill now**
- Build at least 2 AI-integrated projects

**System Design**
- How to design scalable systems
- Load balancers, caching, queues, microservices vs monolith
- This is what senior engineers actually do all day
- Study: Grokking System Design

**Developer Tools Fluency**
- Git deeply (branching, rebasing, PRs)
- Terminal/Linux commands
- Docker basics
- CI/CD pipelines conceptually

---

### The Soft Skills Nobody Talks About
By 2028, these will matter MORE, not less — because AI handles the easy technical parts.

- **Communication** — Can she explain a technical decision to a non-engineer?
- **Problem decomposition** — Breaking a vague idea into small, buildable steps
- **Reading other people's code** — This is a skill almost nobody practices
- **Working with AI tools** — Claude Code, Cursor, Copilot — knowing how to *direct* them correctly

---

## What to Build (Portfolio matters more than a resume)

| Project | What it teaches |
|---|---|
| Full-stack web app with auth | Frontend + backend + DB integration |
| CLI tool in Python | Systems thinking, argument parsing |
| AI-integrated app (use Claude/OpenAI API) | Prompt engineering, API usage |
| A project SHE invented | Problem identification, ownership |

**4 solid projects beats 40 tutorial clones every time.**

---

## Daily Routine (Realistic)

```
Morning (45 min)   → 1-2 LeetCode problems
Afternoon (2 hrs)  → Current learning phase (framework / backend / AI)
Evening (1 hr)     → Build something / work on a project
```

Weekends → go deeper on concepts, not more problems.

---

## What to Avoid

- **Tutorial hell** — watching 40-hour courses and building nothing
- **Chasing every new framework** — depth over breadth always wins
- **Skipping fundamentals** — you can't prompt your way out of not understanding what a pointer is
- **Waiting until final year** — she has a 3 year head start, that's *enormous*

---

## The 2028 Job Market Reality

Companies will hire engineers who can:

1. **Think clearly** about hard problems
2. **Use AI tools fluently** to build faster
3. **Review and verify** what AI produces
4. **Design systems**, not just write functions
5. **Communicate** decisions and tradeoffs

The bar for *writing code* will lower. The bar for *thinking about systems* will rise.

---

## One Sentence Summary

> **She doesn't need to compete with AI — she needs to become the engineer who knows how to use it, verify it, and build things that matter with it.**

She has 3 years and a head start. That's genuinely a gift. The roadmap above, followed consistently, puts her in the top 5% of her graduating class — not because it's magic, but because most people won't follow through.

The only question is consistency. 🎯
